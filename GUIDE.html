✉ Email Validator — Complete Guide

Install, configure, and use the multi-provider email validation app. Covers local dev, Neon PostgreSQL, Vercel deployment, GitHub Actions bulk processing, and full API reference.

1 Overview

Email Validator is a FastAPI web app that validates single emails or bulk CSVs using multiple providers. Results are cached to save API credits on repeat checks.

3 Local Installation

1

Clone the repository

git clone https://github.com/Surya8991/Email-Validator.git
cd Email-Validator
2

Install dependencies

pip install -r requirements.txt

Or with uv: uv pip install -r requirements.txt

3

Create your .env file

cp .env.example .env

Then open .env and fill in at minimum:

BOUNCIFY_API_KEY=your_key_here
4

(Optional) Initialize a Neon database

If you set DATABASE_URL in your .env:

python scripts/init_db.py

Skip this step to use local SQLite instead.

5

Start the server

python -m uvicorn app.main:app --reload

Open http://localhost:8000 in your browser.

Install the pre-push git hook (recommended)

Runs 26 automated checks before every push to prevent broken Vercel deployments:

git config core.hooksPath .githooks

Run manually anytime with: bash scripts/pre_push_check.sh

5 Neon Database Setup

Neon provides free serverless PostgreSQL — required for persistent data on Vercel (Vercel's filesystem is ephemeral).

1

Create a free Neon account

Go to neon.tech and sign up. Create a new project (e.g. "email-validator").

2

Copy the connection string

In your Neon project dashboard, go to Connection Details. Copy the connection string that starts with postgres://. It looks like:

postgres://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
3

Add to your .env

DATABASE_URL=postgres://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
4

Initialize the tables (run once)

python scripts/init_db.py

Output should list 4 tables: job, emailresult, emailcache, apiusage.

✅

The same DATABASE_URL goes into Vercel environment variables AND GitHub repository secrets. All three (local, Vercel, GitHub Actions) share one database.

Migrating an existing database

If you already have a Neon DB from a previous version, run this migration once to add the csv_data column:

ALTER TABLE job ADD COLUMN csv_data TEXT NOT NULL DEFAULT '';

Run it in the Neon SQL editor (project dashboard → SQL Editor).

7 GitHub Actions (Bulk Processing)

Vercel's 10s timeout makes large bulk CSV jobs impossible. GitHub Actions runs the processing with no timeout limit (up to 6 hours).

How it works

  1. User uploads CSV on Vercel → job row created in Neon DB, CSV stored in job.csv_data
  2. Vercel calls GitHub Actions workflow_dispatch API with the job_id
  3. GitHub Actions runner installs deps and runs python scripts/process_job.py --job-id N
  4. Script reads CSV from Neon, validates in chunks of 20, writes results back to Neon
  5. Frontend polls /api/bulk/{id} every few seconds and shows live progress
  6. User downloads results from /api/bulk/{id}/download

Setup steps

1

Create a GitHub Personal Access Token

Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic).
Create a token with workflow scope checked. Copy it.

2

Add the PAT to Vercel env vars

GITHUB_PAT = the token you just created
GITHUB_REPO = YourUsername/Email-Validator

3

Add secrets to your GitHub repo

Go to your repo → Settings → Secrets and variables → Actions → New repository secret:

Secret nameValue
DATABASE_URLYour Neon connection string
BOUNCIFY_API_KEYYour Bouncify key
ℹ️

If GITHUB_PAT is not set, bulk jobs fall back to Vercel's BackgroundTasks. This works locally and on Render/Railway but will hit the 10s limit on Vercel Hobby for large files.

11 Strategies

StrategyCostHow it worksBest for
bouncify_only$Free local syntax+MX pre-filter; if local says invalid, no Bouncify call. Otherwise 1 credit per email.Default. Quick checks; saves credits even on dirty lists.
local_first¢Runs free local checks first (syntax, MX, disposable). Only calls paid API if local doesn't clearly mark it invalid.Dirty lists with lots of obvious invalids — saves 30-50% credits
consensus$$$Calls all enabled providers in parallel. Takes majority vote on verdict.High-value lists where accuracy matters more than cost
waterfall$$Tries local → Hunter → Bouncify → ZeroBounce in order. Stops at first confident result.Balancing accuracy and cost on medium lists
💡

Tip: Use local_first for bulk jobs with large lists. It typically eliminates 20-40% of emails before touching paid APIs, especially for lists with role addresses (info@, admin@) and disposable domains.

12 Cache Settings

Caching stores validation results so repeat checks don't consume API credits. Cache duration is set globally in Settings → Cache Duration.

TTL optionWhen to use
1 dayLists that change frequently; high accuracy requirement
7 daysWeekly email campaigns
30 days (default)Good balance — monthly newsletter lists
90 daysStable lists that rarely change
1 yearArchive / one-time validation only
No cacheAlways re-validate fresh; never store result

The TTL preference is stored in your browser (localStorage) and applied to all validations you run. The server default (CACHE_TTL_DAYS env var) is used as a fallback when no preference is set.

Managing cached entries

  • Browse and search cached results at /cache
  • Delete a single entry: click the delete button in the Cache Browser
  • Purge all expired entries: Settings → Cache Duration → Purge Expired
  • API: POST /api/cache/purge or DELETE /api/cache/{id}

14 Troubleshooting

Internal Server Error on /analytics

🔴

Cause: SQLite-specific SQL was being used on PostgreSQL.
Fix: Already resolved in current codebase — app/routes/ui.py and app/routes/api_stats.py now detect the database dialect automatically.

Vercel build fails: "No 'project' table found in pyproject.toml"

⚠️

Cause: Vercel runs uv lock on any pyproject.toml, even empty ones.
Fix: pyproject.toml must not exist. Config is split into ruff.toml, pytest.ini, mypy.ini.

Vercel build fails: "Function Runtimes must have a valid version"

⚠️

Cause: vercel.json has a "runtime": "python3.12" key.
Fix: Remove the runtime key entirely. Vercel detects Python from .python-version.

Vercel build fails: "Function must contain at least one property"

⚠️

Cause: vercel.json has "api/index.py": {} (empty object).
Fix: Add at least "maxDuration": 10 inside the function config.

Vercel build fails: "maxDuration exceeds maximum"

⚠️

Cause: Hobby plan limit is 10 seconds. maxDuration: 60 requires Pro.
Fix: Set "maxDuration": 10 in vercel.json. Use GitHub Actions for bulk jobs.

Templates not found: "TemplateNotFound"

⚠️

Cause: Relative path "app/templates" in Jinja2Templates constructor.
Fix: Use absolute path: str(Path(__file__).parent.parent / "templates")

Bulk jobs never start / stay at "queued"

⚠️

Cause: GITHUB_PAT or GITHUB_REPO not set, or PAT missing workflow scope.
Fix: Set both env vars in Vercel. PAT must have "Actions (write)" scope. Check /api/health to confirm.

Database connection fails on Vercel

⚠️

Cause 1: DATABASE_URL not set in Vercel env vars — app falls back to ephemeral SQLite.
Cause 2: Tables don't exist yet — run python scripts/init_db.py locally first.
Cause 3: URL missing ?sslmode=require — Neon requires SSL. The app adds SSL automatically.

psycopg2 import error

⚠️

Cause: psycopg2-binary not in requirements.txt.
Fix: Ensure psycopg2-binary>=2.9.9 is listed. Do not use psycopg2 (non-binary) — it requires C compilation.

15 FAQ

Can I use this without Bouncify?

Yes — select "Local" as the only provider. The local stack (syntax, MX, disposable check) is always free and requires no API key. Results are less accurate for edge cases but good enough for basic cleaning.

How many emails can I validate in bulk?

Unlimited — GitHub Actions handles the processing with no timeout. The 10s Vercel limit only applies to the initial upload request, not the validation itself. For very large lists (50k+), the GitHub Actions job may take 30-60 minutes depending on the providers used.

Will re-validating a cached email use API credits?

No. Cached results are returned immediately without calling any provider. The cache duration is set in Settings (default 30 days). Use "No cache" TTL to force a fresh check.

What does "risky" mean exactly?

Risky means the email address technically exists and can receive mail, but it is likely to harm your sending reputation. Examples: catch-all domains (accept all addresses whether they exist or not), disposable/temporary email services (mailinator.com, guerrillamail.com), and role addresses (info@, admin@, noreply@).

Is the local provider accurate enough for production?

For syntax and obvious invalids, yes. For deliverability (will the email actually bounce?), no — you need a paid provider. Use local_first strategy to run local checks first and only call paid APIs for uncertain results.

Can I self-host instead of Vercel?

Yes. The app runs as a standard FastAPI app. Any platform that runs Python works: Railway, Render, Fly.io, a VPS. Remove the api/index.py Mangum entry point and run uvicorn app.main:app directly. For bulk jobs, you can use FastAPI BackgroundTasks instead of GitHub Actions (no timeout limit on non-serverless hosts).

How do I reset/clear all data?

To clear just cache: Settings → Purge Expired, or POST /api/cache/purge.
To clear everything: drop and recreate the tables in Neon's SQL editor, then re-run python scripts/init_db.py.
