Verdict meanings

validEmail exists and can receive mail
invalidEmail doesn't exist or domain has no MX
riskyCatch-all, disposable, or role address
unknownAPI error or inconclusive - not cached

Choose how providers are called per validation. Affects cost and accuracy.

⚡ Bouncify Only $

Single provider, fastest result. Uses 1 Bouncify credit per email. Best for quick checks when you trust Bouncify's accuracy.

🏠 Local First ¢

Runs free local checks first (syntax, MX, disposable). Only calls paid API if local check returns non-invalid. Saves up to 30-50% credits on dirty lists.

🗳 Consensus $$$

Calls all enabled providers in parallel and takes a majority vote. Most accurate - good for high-value lists where false positives/negatives are costly.

🌊 Waterfall $$

Tries providers in order (local → Hunter → Bouncify → ZeroBounce). Stops as soon as a confident verdict is reached. Balances accuracy and cost.

Results are cached - re-validating a cached email costs zero credits regardless of strategy.

Set API keys in .env to enable paid providers. Restart the server after changes.

🏠 Local always on

Free built-in checks: email syntax, domain MX record, disposable domain blocklist (50k+ domains), role-address detection (admin@, info@, etc.), free-provider flag. No API key needed.

Bouncify BOUNCIFY_API_KEY

Primary paid provider. Supports single + bulk (job-based). Maps: deliverable→valid, undeliverable→invalid, accept_all→risky.

bouncify.io ↗
ZeroBounce ZEROBOUNCE_API_KEY

Single + bulk (CSV upload). Includes abuse, spam-trap, and catch-all detection. Maps extra statuses like do_not_mail→risky.

NeverBounce NEVERBOUNCE_API_KEY

Single + bulk (job-based). Granular status: valid, invalid, disposable, catchall.

Hunter.io HUNTER_API_KEY

Single only (no bulk endpoint - uses concurrent individual calls). Includes webmail and disposable detection.

Each provider has an optional daily cap (BOUNCIFY_DAILY_CAP etc.) to prevent runaway credit usage.

View provider status in Settings →

CSV format

email,name,source
john@example.com,John Doe,website
jane@gmail.com,Jane Smith,signup

Column named email is auto-detected. If not found, the first column is used. All extra columns are preserved in the output CSV.

Tips

  • UTF-8 encoding recommended (BOM-safe)
  • Duplicate emails are only validated once - cache handles the rest
  • Progress updates every 20 emails (chunk size)
  • GitHub Actions handles bulk jobs with no timeout limit

Output CSV columns

verdict - final verdict (valid/invalid/risky/unknown)
{provider}_status - per-provider result
from_cache - true if served from cache
All original columns are preserved

Multipart form: file (CSV), providers, strategy

DEL /api/cache/{id} delete one entry
